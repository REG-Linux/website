## Overview

Zelda Classic is a fan-made engine for building Zelda-like quests. REG-Linux exposes the `zc210` metadata group so these `.qst` adventures appear alongside the other `engines` entries, and the `libretro: zc210` core reads your quests, MIDI banks, and map data.

### Quick reference

- **ROM folder:** `/userdata/roms/zc210`
- **Accepted format:** `.qst`
- **Emulator:** `libretro: zc210`
- **Bios folder:** `/userdata/bios/zc210`
- **System group:** `engines`

## BIOS

Drop the Zelda Classic base archive plus optional SoundFont packs under `/userdata/bios/zc210/`. The required files are:

| Filename | Description |
| --- | --- |
| `zcdata.dat` | Core Zelda Classic data |
| `sf2/default.sf2` | General MIDI fallback |
| `sf2/custom*.sf2` | Optional styled sound banks |
| `zcsf.sf2` | Legacy MIDI fallback |

## ROMs

Place your `.qst` quests inside `/userdata/roms/zc210/`. The libretro core accepts zipped `.qst` files as long as the quest sits at the archive root. Keep each questâ€™s `.zcs` map file unique so the menu shows distinct entries.

## Emulator

### libretro: zc210

RetroArch exposes the usual `zc210.*` overrides (`videomode`, `shaders`, `pixel_perfect`, etc.). Use the Quick Menu to remap controls or adjust the display; the core automatically loads whichever quest `.qst` you select from the menu.

## Controls

The default overlay maps South/East/West to the action buttons and the D-pad to movement. Remap buttons via the Quick Menu when quests require keyboard input for text prompts.

## Troubleshooting

- Keep the BIOS files present in `/userdata/bios/zc210/`; missing MIDI banks degrade music fidelity.
- Rename quests if they conflict or keep them in separate subfolders, and ensure each `.qst` includes the `.zcs` map file expected by the engine.
- Visit the generic support pages when RetroArch refuses to detect the BIOS or quest folder.
